## 📚 Übersicht
Das *Assignment Upload Tool* ist ein Prototyp, der für begrenzte [Szenarien](#szenarien) entwickelt wurde. Es ermöglicht:

- **Studierenden** Hausaufgaben hochzuladen, Abgabefristen im Blick zu behalten, Korrekturen einzusehen und den Fortschritt in den einzelnen Kursen zu verfolgen.
- **Tutoren** Korrekturen hochzuladen und ihren eigenen Fortschritt bei der Bearbeitung zu überwachen.

### Besondere Features:
- **Chatbot-KI**: Unterstützt mit Kontextverarbeitung und dreistufiger Fallbacklogik.
- **Kalenderintegration**: Ein integrierter Kalender ermöglicht es Nutzern, ihre Termine stets im Blick zu behalten. Die Termine sind farblich nach Kursen übersichtlich dargestellt.
- **Modernes und interaktives Design**: Bietet klare visuelle Rückmeldungen durch animierte Fortschrittsbalken und einen dynamischen Roboter-Avatar. Unterstützt von LottieFiles zeigt der Avatar Animationen, z. B. während der Bot Nachrichten schreibt.
- **Responsive Design**: Das Tool passt sich an verschiedene Bildschirmgrößen an und ermöglicht eine optimale Darstellung auf Desktops und mobilen Geräten.
- **Internationalisierung**: Unterstützt Deutsch 🇩🇪 und Englisch 🇺🇸 (Übersetzung ist zu ca. 70 % abgeschlossen).

<a id="szenarien"></a>
## 🎯 Szenarien
### Studentenszenario
- Anmelden mit einem beliebigen Benutzernamen und Passwort (außer `tutor`).
- Nach dem Login Benachrichtigungen prüfen.
- Auf die entsprechende Hausaufgabe im Dashboard klicken, um diese einzureichen. *Hinweis: Im Prototyp geht das nur für Hausaufgabe 11 (GMCI).*
- Überprüfen des Fortschritts im Dashboard.
- Den KI-gestützten Hilfeassistenten verwenden, falls Fragen oder Probleme auftreten.
- Verfügbare Korrekturen ansehen.
- Überprüfen des Fortschritts im Dashboard.
- Vor dem Abmelden den Kalender prüfen, um anstehende Termine im Blick zu behalten.
- Abmelden.

### Tutorenszenario
- Anmelden mit dem Benutzernamen `tutor` und einem beliebigen Passwort.
- Nach dem Login Benachrichtigungen prüfen.
- Eine Aufgabe zur Korrektur auswählen, indem auf "Jetzt korrigieren" geklickt wird.
- Die Aufgabe im PDF-Editor mithilfe der Werkzeugleiste *(nicht implementiert)* korrigieren, Punkte vergeben und die korrigierte Version per Mausklick hochladen.
- Überprüfen des Korrekturfortschritts im Dashboard.
- Den KI-gestützten Hilfeassistenten verwenden, falls Fragen oder Probleme auftreten.
- Vor dem Abmelden den Kalender prüfen, um anstehende Termine im Blick zu behalten.
- Abmelden.

<a id="installation"></a>
## ⚙️ Installation

Um das Assignment Upload Tool lokal auszuführen, folgen Sie diesen Schritten:

1. **Repository klonen:**
   ```bash
   git clone https://github.com/differential-topology/assignment-upload-tool.git
   cd assignment-upload-tool
   ```

2. **Webserver starten:**
   Aufgrund von [CORS-Beschränkungen](#cors-beschraenkungen) ist ein lokaler Webserver erforderlich. Wählen Sie eine der folgenden Optionen:

   - **Python (empfohlen Version 3.6 oder neuer)**:
     ```bash
     python -m http.server
     ```

     Falls Port `8000` bereits belegt ist, können Sie einen anderen Port angeben:
     ```bash
     python -m http.server 8080
     ```

   - **Node.js (http-server-Paket)**:
     ```bash
     npx http-server
     ```
     Um einen spezifischen Port zu nutzen, fügen Sie die Option `-p` hinzu:
     ```bash
     npx http-server -p 8080
     ```

    > *Hinweis:* Stellen Sie sicher, dass Python oder Node.js installiert ist. Auf einigen Linux/macOS-Systemen verweist `python` möglicherweise noch auf Python 2. Verwenden Sie in diesem Fall `python3`. Download-Links: [Python](https://www.python.org/downloads/) | [Node.js](https://nodejs.org/).

3. **Zugriff:**
   Öffnen Sie `http://localhost:8000` (oder den angegebenen Port) in Ihrem Browser.

<a id="hilfeassistent"></a>
## 🧠 KI-basierter Hilfeassistent
### Übersicht


Der KI-basierte Hilfeassistent bietet eine Vielzahl von Funktionen, um Nutzer:innen gezielt und kontextbezogen zu unterstützen:

- **Dynamische Kontexterfassung**: Die KI berücksichtigt den bisherigen Chatverlauf, der bei jeder Anfrage aktualisiert und mitgeschickt wird.
- **Benutzerspezifische Daten**: Der Assistent verarbeitet Informationen wie:
  - Benutzername
  - Aktuelle Seite
  - Hochgeladene Dateien (Anzahl, Namen und Typen)
  - Status der Korrektur (für Tutor:innen)
  - Benutzerrolle (Student oder Tutor)
- **Systemprompt**: Ein umfangreicher Systemprompt stellt sicher, dass die KI alle notwendigen Informationen hat, um zielgerichtete und präzise Antworten zu geben.

<details>
<summary><b>Systemprompt anzeigen</b></summary>
  
> **Hinweis:** Um sicherzustellen, dass die Eingabelimits der KI-Modelle nicht überschritten werden, wird bei einer Chatverlaufslänge von mehr als `MAX_CHAT_HISTORY_LENGTH = 2500` Zeichen nur ein relevanter Teil des Verlaufs in den finalen Systemprompt aufgenommen.


### Nutzung
1. Öffnen Sie das Chatfenster durch Klick auf den Roboter-Avatar. Das Chatfenster befindet sich rechts unten in der Ecke (außer auf der Anmeldeseite).
2. Stellen Sie Ihre Frage.  

### Einrichtung
1. Klicken Sie viermal hintereinander auf den Roboter-Avatar, um die Entwicklereinstellungen zu öffnen.  
2. Tragen Sie dort Ihren API-Schlüssel ein. Einen kostenlosen Schlüssel können Sie auf [openrouter.ai](https://openrouter.ai) erstellen:  
    <a id="kostenloser-tarif"></a>
   - **Kostenloser Tarif**: Maximal 20 Anfragen pro Minute und 200 pro Tag.  
   - Unter „Privacy“ können Logging und Model Training deaktiviert werden.  
3. *(Optional)* Wählen Sie ein KI-Modell sowie zwei Fallback-Modelle aus der Liste aus. Standardeinstellungen sind bereits vorkonfiguriert. 

### Fehlerbehebung
- **Keine Antwort**: 
  1. Drücken Sie den Stopp-Button und versuchen Sie es erneut. 
  2. Falls das Problem weiterhin besteht, wählen Sie in den *Entwicklereinstellungen* andere KI-Modelle aus der Liste.
  3. Stellen Sie sicher, dass die Nutzung nicht die Grenzwerte des [kostenlosen Tarifs](#kostenloser-tarif) überschreitet. 
- **Unzuverlässige Modelle**: Viele der von OpenRouter.ai kostenlos zur Verfügung stehenden Modelle funktionieren nicht zuverlässig, insbesondere die High-End-Experimentalmodelle von Google. In den Standardeinstellungen sind daher Modelle vorausgewählt, die sich als verlässlicher gezeigt haben.  
- **Fehlermeldungen**: Bei Problemen gibt der Roboter diese in der Regel direkt als Chat-Nachricht aus. Falls dies nicht ausreicht, können in den *Entwicklereinstellungen* ausführliche Logs über den Logger eingesehen werden.

<a id="probleme"></a>
## 📝 Bekannte Probleme

- **Responsive Design**: Das responsive Design ist zwar grundlegend implementiert, aber nicht optimiert. Auf kleinen Bildschirmen (z. B. Smartphones) ist die Sidebar mit Benachrichtigungen und Kalender nicht sichtbar.
- **Unvollständige Übersetzung**: Die Unterstützung der englischen Sprache ist momentan nur teilweise abgeschlossen.
- **Browser-Kompatibilität**: Optimiert für Safari; andere Browser könnten leichte Abweichungen aufweisen.
- **Flackern**: Insbesondere beim Wechsel zur englischen Sprache treten FOUC und FOTUC auf. Diese Probleme konnten aus zeitlichen Gründen im Prototyp nicht behoben werden.
<a id="cors-beschraenkungen"></a>
- **Lokale Ausführung**: Aufgrund von CORS-Beschränkungen ist ein lokaler Webserver erforderlich.
